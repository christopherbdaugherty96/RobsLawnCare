# Website Implementation Plan

## Purpose
Define the minimum website needed before ShortCut launches publicly.

## Current Phase
Pre-launch planning.

The website should support quote requests, service clarity, trust building, and local visibility. It should not overstate readiness.

## Minimum Website Pages
### Homepage
Purpose:
- explain ShortCut
- show service area
- list grass cutting services
- mention power washing as planned only if readiness is not complete
- drive visitors to request a quote

### Services Page
Purpose:
- explain grass cutting services
- explain seasonal cleanup
- explain power washing only when ready or label it as planned

### Quote Page
Purpose:
- collect quote requests
- gather property details
- request photos when useful
- set expectations for response time

### Gallery Page
Purpose:
- show before/after work once real photos exist
- avoid stock-photo overclaiming

### Contact Section
Purpose:
- provide simple contact method
- reinforce service area
- avoid unverified claims

## Website Truth Rules
Do not claim:
- fully operating if pre-launch
- licensed unless confirmed
- insured unless confirmed
- commercial services unless ready
- power washing availability unless ready

## Launch Website Checklist
- [ ] domain selected
- [ ] homepage copy finalized
- [ ] service page copy finalized
- [ ] quote form ready
- [ ] contact method ready
- [ ] Google Business Profile aligned
- [ ] before/after gallery structure ready
- [ ] mobile layout checked
- [ ] service area visible
- [ ] claims verified

## Simple CTA
Use one primary call to action:

Request a Free Quote

## Operating Rule
The website should help convert local customers, not pretend the business is larger or more established than it is.
